Getting Started

Getting your API Key

Before you start using the Google Books API, you need to get an API Key. This is necessary in order to perform the searches and actually use the Google API. As discussed on the next page, the API Key has to be appended onto all search queries. This page will help you get your API Key.

Google Developer

To get your API Key, you need to go to the Google Developer Console.

Once at the Google Developer Console, you will need to start a new project (if you do not have one already created that you want to use). To create a new project, simply click the 'New Project' button on the top left of the page. This should pop up the screen shown below.

Give your project a name, and you can use the automatically generated Project ID, noting that this needs to be unique. Then click 'Create'.

Now we need to enable the API Key for the Books product. Go ahead and click on the project you just created. On the new apge you are on, you should click the 'APIs & auth' link on the left sidebar. This will drop down a few more items to choose from. Click on 'APIs' and search for 'Books' on the new page. You should have something called 'Books API' in the results - click on that. Finally, click 'Enable API' and you should now have this screen below (if already enabled, skip this step):

Your Google Books API Key is now enabled! Unfortunately, we are not yet done. Now click on 'Credentials' in the left sidebar. You should have two choices - oAuth or Public API access. As mentioned on the first page of this guide, we are going to be using solely public access. So click the button entitled 'Create new Key' under the 'Public API access' section. You will have choices on what kind of key to create. This guide will use the 'Browser Key', so create that one for now. Now click 'Create' and you have an API key!

You are now ready to create searches for the Google Books API.
